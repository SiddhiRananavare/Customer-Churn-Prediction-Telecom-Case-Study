# Customer-Churn-Prediction-Telecom-Case-Study
Developed a customer churn prediction model to identify customers likely to leave. Performed data preprocessing, EDA, and feature engineering. Built and evaluated classification models to improve accuracy and provided insights to support customer retention strategies.

📌 Customer Churn Prediction – Telecom Case Study

🔹 Overview

-This project focuses on predicting customer churn in a telecom company using machine learning techniques. The goal is to identify customers likely to leave and provide insights to improve retention strategies.

🔹 Objectives

-Predict whether a customer will churn

-Identify key factors influencing churn

-Support data-driven customer retention decisions

🔹 Dataset Description

-The dataset contains customer-related information such as demographics, service usage, billing details, and contract information. The target variable indicates whether the customer has churned (Yes/No).


🔹 Key Tasks Performed

-Data cleaning and preprocessing

-Handled missing values and encoded categorical features

-Conducted exploratory data analysis (EDA)

-Performed feature engineering

-Split data into training and testing sets

-Built classification models (Logistic Regression, Decision Tree, Random Forest)

-Evaluated model performance using multiple metrics

🔹 Tools & Technologies
-Python

-Pandas

-NumPy

-Scikit-learn

-Matplotlib / Seaborn

🔹 Evaluation Metrics
-Accuracy

-Precision

-Recall

-F1 Score

-Confusion Matrix

🔹 Results

-The model successfully identified high-risk customers and achieved good predictive performance, helping in proactive decision-making.

🔹 Conclusion

-This project demonstrates how machine learning can be used to predict customer churn and provide actionable insights to reduce customer loss and improve business performance.
